# Driver-Pass-system-design-
This is for a system design project 
The client Driver Pass needed a new system to provide a user interface so users can take practice tests, view course content, and make appointments. They wanted a new system that will increase the probability that the students pass their driver license test the first time.
I think in this project I incorperated adequate diagrams that effectivly portray system requierments. The part that I could improve is within the system design document, the technical requirments is lacking sufficient detail and could use more work. 
I determined from the Driver Pass interview the different user needs. In my diagrams I showed what those user needs were and how they relate to the system as a whole. It is important to consider what the user needs are because to have a succesful system, it needs user input so that the users are satisfied with the final product.
My approach to designing a system was to determine all of the actors and use cases. With this information readily available I could complete most design documents. I used this information and other information from the Driver Pass intervierw to determine what data and objects are needed to make this a succesful system. I used that information to create a class diagram. I think writing down the necessary information required for diagrams first, puts everything in perspective that allows me to make accruate diagrams.
